Always add a .gitignore file to project directory before initial commit

.gitignore file should have these three lines:

node_modules/
.DS_Store
dist/

We need to create the .gitignore file and specify which files shouldn't be tracked before committing.

To stop git from tracking a file first add that file to the .gitignore file
next, run "git rm -r --cached [FILE-NAME]",
where [FILE-NAME] is the name of the file Git will no longer track

Node package manager

or just npm

"npm init -y"
This creates a file called package.json in the project's root directory.

This is the webpack documentation.

"npm install webpack@4.39.3 --save-dev --save-exact"
This will install webpack and add the dependency to our package.json file.

**The "--save-exact" flag ensures that we save the exact version of webpack to our dev dependencies.
**the "--save-dev" flag specifies that we want webpack to be a development dependency
**All of the dependencies we install with the "--save-dev" flag will be included in devDependencies.

"npm install webpack-cli@3.3.8 --save-dev"
This package will allow us to use webpack from the command line.

"npm install"
This will automatically add the correct Node modules to the project.

"npm prune"
The command will ensure only dependencies listed in the manifest are actually installed (while others are removed).

Webpack uses a dependency graph to recursively manage an application's assets

We commonly use main.js as an entry point, where we include our interface logic.
Webpack needs this entry point in order to recursively gather all the other files

make a file in top level of project like this
"webpack.config.js"
const path = require('path');

module.exports = {
entry: './src/main.js',
output: {
filename: 'bundle.js',
path:
path.resolve(__dirname, 'dist')
}
};

"import" is a way to import a piece of code from one file into another.
Used in the top lines of the entry point, aka main.js

"export" is a way to export a piece of code from one file into another.
Used in the top lines of a buisness logic page, aka function-logic.js
